# LLM Translator Test

Исследование и разработка локальных офлайн-переводчиков на базе LLM и специализированных NMT-моделей.

---

## Цель проекта

Найти и/или собрать лучший локальный переводчик (приоритет: RU↔EN), который:
- работает **полностью офлайн**
- **сохраняет форматирование** документов (PDF searchable, DOCX, XLSX, MD, TXT)
- пригоден для **слабых ПК** (CPU-only, ≤16 GB RAM)
- не требует платных сервисов или сложной инфраструктуры

---

## Правила проекта

- Стороннее ПО (Ollama, LM Studio и аналоги) — **без явного согласования не ставим**
- `pip install` и `git clone` — разрешены
- Java — разрешена (нужна для Okapi Framework)
- Скачивание моделей — разрешено (подключаем к существующему или собственному коду)
- Форматирование документов **обязательно сохраняется** для всех типов файлов
- Если готовое решение побеждает — берём его как есть, без пересборки

---

## Два параллельных трека

### Трек 1 — Бенчмарк моделей и стеков
**Ветка:** `master`
**Железо:** RTX 4080 (16 GB VRAM). Модели тяжелее 13B тестируются параллельно на отдельном ПК.

Сравниваем качество, скорость и ресурсозатраты моделей и готовых стеков.
Подробнее: [ПЛАН 1 — Бенчмарк моделей и стеков.md](ПЛАН%201%20—%20Бенчмарк%20моделей%20и%20стеков.md)

### Трек 2 — LightLocalTranslator
**Ветка:** `light-local-translator`
**Железо:** CPU-only, 8–16 GB RAM (целевое).

Офлайн-переводчик для слабых ПК. Стратегия: сначала готовые решения — если одно побеждает, берём как есть. Только если не находим — собираем своё.
Подробнее: [ПЛАН — LightLocalTranslator.md](ПЛАН%20—%20LightLocalTranslator.md)
